## Problem

Starting a serious React project often begins with an hour of chores: routing, theming, i18n, metadata, fallback pages, linting, formatting, and the first pass at an app shell. Forge came from wanting that setup to feel less like copying yesterday's project.

## Solution

I built a CLI that creates a small starter first, then layers in framework-aware overlays and optional feature packs. It can add theme switching, English and Arabic routing, framework-specific i18n, starter docs, sound hooks, and lint/format setup.

## Architecture

Forge is a single-package TypeScript CLI built around scaffold adapters, framework overlays, and feature packs. It can generate starters for Next.js, Vite, and TanStack Start, with Base UI or Radix-based surfaces through shadcn scaffolds. The default preset is intentionally opinionated: Next.js, Base UI, LTR, pnpm, and Biome.
